The 80/20 Guide to Core JavaScript Concepts

This guide provides an efficient summary of the most critical, foundational concepts in JavaScript. By focusing on the 20% of principles that yield 80% of the results, we concentrate on the core mechanics you will use every day to write functional, maintainable, and performant code.

1. The Foundation: Integrating and Debugging

Before writing complex logic, a developer must master how to add JavaScript to a page and how to use the browser's console to diagnose problems.

1.1. Integrating JavaScript Code Efficiently

Benefit Impact
Code Separation Makes files cleaner and easier to navigate.
Readability Simplifies updating and debugging code over time.
Caching Browsers store external JS files for faster subsequent loads.

A critical concept is non-blocking JavaScript. Use the defer attribute in your <script> tag to ensure scripts don't delay the visual rendering of the page.

1.2. The JavaScript Console

The Console is the primary channel of communication between your code and you. It serves two main functions:

  • Displaying Information: Use console.log() to inspect variables in real-time.
  • Reporting Errors: It provides descriptive messages and line numbers when code fails.

2. Storing Information: Variables and Data Types

2.1. Declaring Variables: let vs. const

Modern JavaScript uses let and const. Recommendation: Use const by default. Only use let when you know a value must change.

  • let: For values that change (e.g., game scores).
  • const: For constants (e.g., date of birth).

2.2. Core Data Types

Data Type Description
string Text characters enclosed in quotes.
number Integers and decimals.
boolean Logical true or false.
null Intentional absence of a value.

2.3. Template Literals

Use backticks (`) and the ${variable} syntax to embed data directly into strings:

const customerFirstName = "Marie";
const accBalance = 10;
console.log(`Hi ${customerFirstName}. Your balance is €${accBalance}.`);

3. Manipulating Data

3.1. Numeric Operations

JavaScript follows standard mathematical rules (PEMDAS). Use parentheses () to control the order of operations.

5 + 2 * 3;   // 11
(5 + 2) * 3; // 21

3.2. String Principles

Strings are zero-indexed and immutable. Methods like toUpperCase() return a new string; they do not change the original.

let msg = "hello";
msg = msg.toUpperCase(); // Must reassign to save changes
console.log(msg); // "HELLO"

4. JavaScript as a Loosely-Typed Language

JavaScript determines data types automatically at runtime. This offers flexibility for rapid development but increases the risk of errors if a variable changes type unexpectedly.

5. Core Principles Recap

  • Prioritize External Scripts: Use defer for performance.
  • Master the Console: Use it to inspect and debug constantly.
  • Default to const: Create predictable code.
  • Remember Immutability: Always reassign modified strings.
  • Respect Precedence: Use parentheses for math clarity.
  • Use Template Literals: For readable string concatenation.
